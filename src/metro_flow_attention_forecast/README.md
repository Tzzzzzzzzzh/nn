# 地铁客流时序预测与注意力加权优化

本项目实现了一个面向城市轨道交通的地铁站客流时序预测模块。客流预测是智能交通调度中的基础任务，可以辅助地铁站提前安排安检通道、站台引导、列车加开和限流策略。传统季节性基线只使用上一周同一时间段的客流，遇到天气波动、活动客流和短期趋势变化时误差较大。本项目构建 15 分钟粒度的合成客流序列，并实现一个注意力加权的时间滞后预测模型，用于提升短期客流预测精度。

## 功能内容

- 构建 14 天、15 分钟粒度的地铁站客流时序数据。
- 模拟早高峰、晚高峰、午间客流、周末变化、天气扰动和活动冲击。
- 实现上一周同时间段 `seasonal_baseline` 基线预测策略。
- 实现 `attention_temporal_forecast` 注意力加权预测策略，融合最近滞后、日周期滞后、周周期滞后、天气和活动上下文。
- 使用岭回归读出层对注意力聚合特征进行拟合，避免依赖大型深度学习框架。
- 统计 MAE、RMSE、MAPE、拥挤预警召回率和精确率。
- 生成预测曲线、误差分布、注意力权重热力图和指标对比图。
- 输出 `forecast_result.csv` 和 `metrics.json`，方便复现实验和写提交说明。

## 运行方法

在本目录执行：

```bash
python main.py
```

也可以指定输出目录：

```bash
python main.py --output assets
```

运行后会生成：

- `forecast_curve.png`：真实客流、季节性基线预测和注意力预测曲线。
- `error_distribution.png`：两种预测策略的误差分布。
- `attention_weights.png`：测试集中一天内不同时间滞后特征的注意力权重。
- `metric_comparison.png`：MAE、RMSE、MAPE 和拥挤预警召回率对比。
- `forecast_result.csv`：测试集逐时间点预测结果。
- `metrics.json`：核心指标和优化百分比。

## 项目意义

该模块贴近神经网络课程中的时间序列建模与注意力机制思想。虽然没有使用大型深度学习框架，但模型保留了“多时间滞后特征加权融合”的核心结构，能够解释模型在不同时间段更依赖最近趋势、日周期还是周周期信息。该方法可以继续扩展为 LSTM、GRU、Transformer 或多站点图神经网络客流预测。

## 测试

```bash
python -m pytest tests
```

测试内容包括：

- 注意力预测是否降低 MAE、RMSE 和 MAPE。
- 注意力权重是否满足概率归一化。
- 主程序是否能生成指标文件和全部可视化图片。
