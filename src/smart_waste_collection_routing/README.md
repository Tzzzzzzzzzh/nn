# 智慧垃圾桶收运路径与溢出风险优化

本项目实现了一个面向城市环卫系统的智慧垃圾桶收运路径优化模块。传统收运车辆通常按照距离最近或固定线路进行清运，但不同区域的垃圾桶填充速度不同，商业区、学校、居民区和公园的溢出风险也不一样。如果只追求行驶距离最短，可能会错过高优先级垃圾桶，导致溢出投诉和二次清扫成本增加。本项目在最近邻基线策略基础上，实现融合填充率、增长速度、优先级和行驶距离的紧急度感知路径规划。

## 功能内容

- 构建居民区、学校、商业区和公园四类区域的智慧垃圾桶场景。
- 为每个垃圾桶设置坐标、当前填充率、填充增长速度、服务时间和优先级。
- 设置单车班次时长、车辆容量和平均行驶速度约束。
- 实现最近邻 `nearest_neighbor` 基线收运策略。
- 实现紧急度感知 `urgency_aware` 优化策略，综合考虑预计到达填充率、溢出风险、优先级和行驶距离。
- 统计服务垃圾桶数量、总行驶距离、收运填充量、溢出数量、平均溢出风险、高优先级服务数量和路线效率评分。
- 生成路线地图、溢出风险时间序列、区域服务数量图和指标对比图。
- 输出 `route_schedule.csv` 和 `metrics.json`，方便复现实验和写提交说明。

## 运行方法

在本目录执行：

```bash
python main.py
```

也可以指定输出目录：

```bash
python main.py --output assets
```

运行后会生成：

- `route_map_comparison.png`：最近邻路线和紧急度感知路线对比。
- `overflow_risk_timeline.png`：优化路线中每个服务点的到达填充率和溢出风险。
- `district_service_counts.png`：不同区域被服务的垃圾桶数量对比。
- `metric_comparison.png`：行驶距离、溢出数量、风险和效率指标对比。
- `route_schedule.csv`：两种策略下的服务顺序、到达时间和风险。
- `metrics.json`：核心指标和优化百分比。

## 项目意义

该模块展示了智慧城市中的动态资源调度思想。垃圾收运不能只看“哪一个最近”，还要考虑“哪一个最快溢出”和“哪一个影响最大”。紧急度感知路线可以让车辆优先处理高风险点位，降低垃圾桶溢出概率，同时保持路线在班次时长和车辆容量约束内可执行。

该项目可以继续扩展到多车辆协同、实时传感器数据接入、垃圾分类桶、道路拥堵、动态投诉热度和碳排放评估。

## 测试

```bash
python -m pytest tests
```

测试内容包括：

- 紧急度感知路线是否降低平均溢出风险。
- 路线是否满足车辆容量和班次时长约束。
- 主程序是否能生成指标文件和全部可视化图片。
